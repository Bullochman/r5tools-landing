Privacy Policy

Last updated: July 2026. Short version: we don't collect what we don't need. Long version below.

TL;DR: Your roster data + Discord webhooks stay private. We don't sell anything. Photos in Profile Studio never leave your device. Analytics are aggregate + IP-masked.

What we collect

  • Access-code redemptions — which code you entered, when, from which tool. Stored in a per-code counter.
  • IP address — masked to last-octet-x for analytics; unmasked briefly for rate limiting (auto-forgotten after 1 hour).
  • Browser language — for auto-toggling UI language.
  • Country — inferred from IP via Cloudflare's CF-IPCountry header, only for analytics ("how many R5s from KR are using this today").
  • Roster data — CSVs you upload to Roster Extractor. Stored per alliance signature (SHA-256 hash of your R5 name + top 8 members), separate from IP. Never shared with other alliances.
  • Discord webhook URLs you paste — stored server-side under your alliance access code so sibling tools can auto-sync. Never exposed to other alliances.
  • Support request contents — the text you type in the 💡 Feature Request modal. Forwarded to Evan's Discord + stored server-side by reference ID.

What we DON'T collect

  • Real names
  • Email addresses (unless you explicitly use the email-magic-link login path)
  • Phone numbers
  • Payment card info — Stripe handles that; we only see the tier + code minted
  • Screenshots or photos beyond the specific tool you're using (Profile Studio processes 100% in your browser — never uploads)
  • Any in-game credentials (username, password, session token)

Cookies

  • lws_unlock_code — your access code, cross-subdomain on .r5tools.io, 90-day expiry. HttpOnly, Secure, SameSite=Lax.
  • lws_email_session — HMAC-signed session token if you used email magic-link auth. 90-day expiry.
  • No third-party trackers, no Google Analytics, no Facebook Pixel, no advertising cookies. Ever.

localStorage

  • lws_discord_webhook — your alliance's Discord webhook URL (per-tool origin).
  • lws_support_requests — list of ref IDs of your support requests, so the widget can show status without needing an account.
  • lws_alliance_roster — cached CSV of your latest extracted roster, for cross-tool carry-through.
  • lws_current_warzone — your warzone selection.
  • You can clear all of this at any time via your browser's Clear Site Data option.

Sharing with third parties

  • Stripe — handles all payment processing. We never see card details.
  • Anthropic (Claude API) — the KB Chat + Roster Extractor call the Claude API. Anthropic sees the KB context + your questions / roster frame images, subject to Anthropic's data usage policy (they don't train on API traffic by default).
  • Cloudflare — DNS + edge for r5tools.io. Standard edge logs.
  • Railway — hosting for our backend. Standard server logs.
  • GitHub Pages — hosting for the static tools. Standard access logs.
  • That's it. No advertisers. No data brokers. No re-sale.

Your rights

  • Delete your data: email hello@r5tools.io with your alliance access code. We'll wipe the roster snapshots + webhook + support tickets tied to it within 7 days.
  • Export your data: same email. We'll send you a JSON dump of everything tied to your code.
  • Correct wrong data: use the tool's Edit UI (Roster Extractor lets you edit rows before saving).

Children

This tool is for players of a mobile military strategy game. It's not designed for children under 13. If we discover data belonging to a child under 13 we delete it.

Changes

Material changes to this policy get in-tool notification 7 days before taking effect. Current version always at r5tools.io/privacy.html.

Contact

hello@r5tools.io — usually answered within a few hours.
